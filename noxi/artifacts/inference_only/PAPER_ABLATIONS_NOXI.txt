PAPER_ABLATIONS_NOXI
====================

Purpose
-------
This note documents the clean NOXI / NOXI-J feature-ablation runs created on
2026-06-29 for the paper. The goal was to rerun the final submitted-style
architecture while varying only the input feature set, so the ablation table can
be interpreted as a feature-family comparison rather than a mixture of older
architectures.

These runs are validation ablations, not the literal submitted checkpoint
directories. Architecturally, however, they match the clean final NOXI/NOXI-J
model family: causal dyadic shared-head TCN, gated multimodal fusion,
domain_role metadata, output calibration, W750 windows, and VideoMAE visual
features.

Architecture Notes
------------------
The clean final NOXI/NOXI-J runs use dyadic input and a shared dyadic prediction
head.

In this setup, novice and expert features are fused per modality and then paired
as dyadic input. The TCN hidden states are therefore conditioned on both
participants. The shared prediction head outputs two channels, corresponding to
novice and expert predictions.

The metadata setting is domain_role with output calibration. Metadata is not
injected into the temporal encoder. Instead, it calibrates the output via a
small metadata-conditioned scale/bias readout.

Model Variants
--------------
self-only
  Backbone: simple

  Novice and expert streams are processed independently with the same
  person-level TCN. Each role prediction uses only that role's own fused
  audio/text/visual features. There is no partner information path before the
  output. Metadata still applies the same output-calibration scale/bias used in
  the submitted-style runs.

dyadic shared
  Backbone: dyadic_shared

  This is the final submitted-style dyadic model family. Audio/text/visual
  features are first projected and gated separately per role. The fused novice
  and expert streams are then concatenated as a dyadic input to one temporal
  encoder. A shared two-channel prediction head emits novice and expert
  engagement predictions. Partner information is represented implicitly through
  the shared dyadic hidden state.

gated partner
  Backbone: gated_partner

  Novice and expert are encoded separately with a shared role encoder. For each
  target role, the model computes a learned gate over the partner hidden state.
  The prediction head receives the target hidden state plus the gated partner
  hidden state. This is explicit partner conditioning, but the amount of partner
  information is controlled by a learned gate.

attention sepenc
  Backbone: attention

  "sepenc" means separate encoders. Novice and expert each have their own TCN
  encoder weights. Each role then attends over a joint self+partner temporal
  context using the role-specific attention module. The attention is causal and
  uses the configured past-frame window. The prediction head receives the role's
  own hidden state plus the attended context. This is explicit self/partner
  temporal attention with separate role encoders.


Metadata Handling
-----------------
All submitted-profile partner/self ablations use:

  --metadata-set domain_role
  --metadata-injection output_calibration

Metadata is not concatenated into the modality gate, TCN input, or standard
prediction-head input. Instead, it is encoded by a small metadata branch and
used after the raw two-role prediction:

  raw prediction = TCN/head(fused dyadic or role features)
  metadata       = domain_role metadata for novice and expert
  final          = scale(metadata) * raw prediction + bias(metadata)

The scale is conservative in code:

  scale = 1 + 0.1 * tanh(scale_logits)

The metadata branch therefore provides a learned role-specific calibration of
the raw prediction, not a separate temporal representation.


Code Notes for Reproducibility
------------------------------
Before running these ablations, the model wrapper was extended so the simple
and partner backbones could use the same submitted-style settings:

  /work/ACM/ACM-clean/src/acm_pipeline/models_tcn.py

The changes were intentionally narrow:

  1. TCNRegressor and IndependentDyadicTCNRegressor accept causal_tcn.
  2. The simple backbone supports metadata_injection=output_calibration.
  3. DyadicRoleHeadTCNRegressor, GatedPartnerTCNRegressor,
     SharedEncoderRoleAttentionTCNRegressor, and RoleAttentionTCNRegressor
     accept causal_tcn.
  4. A generic MetadataOutputCalibratedRegressor wraps non-dyadic-shared
     partner backbones for output calibration.

This keeps the partner variants comparable to the submitted W750 profile
without changing their core partner/self modelling logic.

Interpretation
--------------
Within single-corpus training, explicit partner modelling helped both NOXI and
NOXI-J. For NOXI-only, gated partner was strongest, with attention sepenc close
behind. For NOXI-J-only, gated partner was also strongest.

In joint training, dyadic shared remained the strongest or near-strongest
overall submitted-style architecture. For S375, dyadic shared had the best
overall CCC, while attention sepenc gave the highest NOXI-specific CCC but a
lower NOXI-J CCC. For S563, dyadic shared gave the strongest overall and
NOXI-J-specific CCC. This supports the paper interpretation that lightweight
explicit partner conditioning can help in single-corpus regimes, while the
joint NOXI+NOXI-J setting still favors the shared dyadic submitted-style model.
